\begin{frame}{Probes}
We can project activations at any point in the forward pass along any direction,
and thereby classify the input on a linear scale.

\note{
\begin{itemize}
\item We can just project the activations at any point in the forward pass along any direction, and thereby classify the input of the transformer on a linear scale
\item If the direction is a feature with some human-understandable meaning, we would expect it could classify inputs that have this meaning (a Spanish feature could classify Spanish text)
\item Or alternatively classify model cognition (a ``lying'' feature when the model decides to output a lie)
\end{itemize}
}

\begin{frame}{Ablating Features \cite{arditi2024}}
\textbf{Activation addition:}
\[ x^{(l)\prime} \leftarrow x^{(l)} + r^{(l)} \]

\vspace{0.5em}
\textbf{Direction ablation:}
\[ x' \leftarrow x - \hat{r}\hat{r}^\top x \]

\vspace{0.5em}
\textbf{Weight orthogonalization:}
\[ W'_{\text{out}} \leftarrow W_{\text{out}} - \hat{r}\hat{r}^\top W_{\text{out}} \]

\vspace{0.3em}
{\small $r$: feature direction \quad $x$: activation \quad $W$: weight matrix}

\note{
\begin{itemize}
\item Once we have a feature, we can do more than just adding it somewhere to see what happens
\item We can also ablate it out, and make a model basically blind to that concept
\item We can fold this projection into the weights: basically making the model blind to a concept in a way that is not easy to detect from the weights alone
\end{itemize}
}
\end{frame}

\begin{frame}{Superposition \cite{elhage2022}}
\begin{center}
\includegraphics[width=0.5\textwidth,height=0.7\textheight,keepaspectratio]{figures/elhage2022/fig10.png}
\end{center}

\note{
\begin{itemize}
\item The reason we get polysemantic directions: the model has more features than dimensions in its activation space
\item If you have 2 dimensions and want to encode 2 properties (Portuguese, Shakespearean), you can perfectly encode both
\item If you want a third (ozone hole), you can still encode all 3, but with some noise
\item If the outer two features are active, this looks like the third feature
\item So a Portuguese Shakespeare text is indistinguishable from ozone hole
\item This is not so bad if features are sparse: Portuguese and Shakespearean rarely co-occur
\item But for always-present features (age, height, wealth), superposition would be quite bad
\item So superposition makes sense as long as features are sparse
\end{itemize}
}
\end{frame}

\begin{frame}{Johnson--Lindenstrauss Lemma \cite{johnson1984}}
In $N$ dimensions, we can pack
\[ m \leq e^{c \cdot \varepsilon^2 \cdot N} \]
vectors with pairwise dot product at most $\varepsilon$.

\vspace{1em}
Features scale \textbf{exponentially} with dimensions, given overlap $\varepsilon$.

\note{
\begin{itemize}
\item The effect is quite big: instead of scaling the number of features linearly with dimensions, we scale exponentially given a feature overlap of $\varepsilon$
\end{itemize}
}
\end{frame}

\begin{frame}{Dictionary Learning \cite{arora2013}}
\textbf{Setup:} Unknown dictionary $A = [A_1, \ldots, A_m] \in \mathbb{R}^{n \times m}$ with $m > n$.

We observe samples $y = Ax$, where $x \in \mathbb{R}^m$ is $k$-sparse.
We know neither $x$ nor $A$.

\vspace{0.5em}
\textbf{Claim:} $A$ is recoverable up to permutation and sign, in polynomial time, if:
\begin{itemize}
\item \textbf{Sparsity:} $k \leq c \cdot \min\!\left(m^{2/5},\; \frac{\sqrt{n}}{\mu \log n}\right)$
\item \textbf{Incoherence:} $|\langle A_i, A_j \rangle| \leq \frac{\mu}{\sqrt{n}}$ for all $i \neq j$
\item \textbf{Sufficient data:} $p = \Omega\!\left(\frac{m^2}{k^2} \log m\right)$ samples with varying support
\item \textbf{Well-behaved coefficients:} independent, symmetric, bounded away from zero
\end{itemize}

\note{
\begin{itemize}
\item This gives us theoretical backing for why finding features in superposition should even be possible
\item The setup: imagine a set of directions (features), and we observe weighted sums of a few of them at a time
\item We never know which features are active or what the weights are, we just see the resulting vector
\item The claim: under these conditions, the set of directions is uniquely identifiable
\item There is no alternative set of directions and weights that could produce the same observations
\item Sparsity: not too many features active at once
\item Incoherence: the true feature directions are not too similar to each other
\item We need enough data with varying combinations of active features
\item Importantly, this does not tell us that SAEs specifically will find these features
\item It tells us the features are there to be found, and that some polynomial-time algorithm exists
\item Whether gradient descent on the SAE objective converges to the right answer is a separate question
\end{itemize}
}
\end{frame}

\begin{frame}{Compressed Sensing \cite{candes2004,donoho2006}}
\textbf{Setup:} Known dictionary $A \in \mathbb{R}^{n \times m}$.
We observe $y = Ax$, where $x$ is $k$-sparse.
We want to recover $x$.

\vspace{0.5em}
\textbf{Problem:} $m > n$, so there are infinitely many solutions to $Ax = y$.

\vspace{0.5em}
\textbf{Claim:} If $A$ satisfies the Restricted Isometry Property (RIP), then
\[ \min_z \|z\|_1 \quad \text{s.t.} \quad Az = y \]
recovers $x$ exactly.

\vspace{0.5em}
$\ell_1$ minimization finds the sparse solution; $\ell_2$ minimization does not.

\note{
\begin{itemize}
\item Once we know the dictionary, there is a second question: for a given observation, which features are active and how strongly?
\item This is an underdetermined system: more unknowns than equations, so infinitely many solutions
\item Key insight: if we additionally require the solution to be sparse, it becomes unique
\item We can find it by minimizing the $\ell_1$ norm, which is a convex problem
\item Why $\ell_1$ and not $\ell_2$? The $\ell_1$ ball has sharp corners on the axes, so the optimizer lands where most coordinates are zero
\item The $\ell_2$ ball is round, so $\ell_2$ minimization spreads energy across all coordinates
\item This is exactly what the sparsity penalty in SAEs does
\item Compressed sensing tells us that under the right conditions, the sparse decomposition is unique
\end{itemize}
}
\end{frame}

\begin{frame}{Sparse Autoencoders \cite{bricken2023}}
\[ f = \text{ReLU}(W_{\text{enc}}(x - b_{\text{dec}}) + b_{\text{enc}}) \]
\[ \hat{x} = W_{\text{dec}} f + b_{\text{dec}} \]
\[ \mathcal{L} = \underbrace{\|x - \hat{x}\|_2^2}_{\text{reconstruction}} + \lambda \underbrace{\|f\|_1}_{\text{sparsity}} \]

\note{
\begin{itemize}
\item The activations of the transformer are blown up to a larger number of features
\item This incentivizes the SAE to represent all the information, but represent it sparsely
\item We saw earlier that there is only ``one way'' to sparsely represent data, under various assumptions
\item So this is supposed to give us ``true features'' from the superposition mess
\item We need to decide the dictionary size beforehand, so we cannot be sure a feature is absent just because we did not find it
\item Unlike probes, this is unsupervised: we do not decide what features to find
\item Now we get features, but how do we find out what they mean?
\item Question to audience: what would convince you that a feature means ``Arabic''?
\end{itemize}
}
\end{frame}

\note{

\begin{frame}{Comparing SAEs \cite{karvonen2025b}}
\paperfig{figures/karvonen2025b/fig1.png}{03_transformer_features/figures/karvonen2025b/fig1-caption.txt}

\note{
\begin{itemize}
\item SAEs are not classical ML where we just want low loss on an eval set
\item We actually want them to be a good interpretability tool, and that is hard to quantify
\item Loss recovered: what percentage of the transformer performance is recovered when we replace activations with the SAE reconstruction vs.\ delete the activations
\item Auto interpretability: give an LLM examples of when the feature was active/not active, it proposes an explanation, then predicts feature activation on a test set
\item Mean absorption score: for hierarchical features like starting letters and words, how much are rarer features absorbed into more common ones?
\item SCR top 20: train a classification probe on data with spurious correlations (e.g.\ profession and gender), then ablate the gender feature --- does the bias decrease?
\item K-sparse probing: take a concept like sentiment, train a probe on the single most relevant feature --- what is the probe's performance?
\item Extremely mixed results: different SAE architectures have different strengths and weaknesses
\end{itemize}
}
\end{frame}

\begin{frame}{Safety-Relevant Feature Steering \cite{anthropic2025b}}

\vspace{0.5em}
\begin{enumerate}
\item Find \textbf{evaluation awareness} features via:
\begin{itemize}

\item SAE features (discriminating eval transcripts)
\end{itemize}
\item \textbf{Inhibit} these features (negative steering at 25\% of activation norm)
\end{enumerate}

\vspace{0.5em}
\textbf{Results:}
\begin{itemize}
\item Reduced verbalized evaluation awareness
\item Modestly increased misaligned behavior
\item Random feature steering also had a (weaker) effect
\end{itemize}

\note{
\begin{itemize}
\item The chain of thought said less things like ``I know this is a test''
\item The model behaves less aligned, but not in the most egregious cases of previous models (blackmail, sabotage)
\item Some of the effect is just the model being pushed out of distribution, since random features also cause changes
\item They used both contrastive vectors (subtracting activations on eval vs.\ non-eval prompts) and SAE features identified by discriminating eval-aware vs.\ non-eval-aware transcripts
\end{itemize}

\note{
\begin{itemize}
\item We can not only vary the architecture and loss of the SAE itself, but also what kind of data we are acting on
\item Transcoders do not sparsify a single layer of activations, but reconstruct the next layer from the current layer
\item The idea: sparsify the computation going on in some part of the transformer
\item Crosscoders take multiple layers and reconstruct multiple layers out of multiple layers
